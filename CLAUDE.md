# CLAUDE.md

このファイルは、Claude Code (claude.ai/code) がこのリポジトリで作業する際のガイダンスを提供します。

## 共通コマンド

### 開発
- `npm start` - Electronアプリケーションを起動
- `npm run dev` - 開発モードで起動
- `npm install` - 依存関係をインストール

### ビルド
配布用にアプリケーションをビルドするには、electron-builderを追加する必要があります：
```bash
npm install --save-dev electron-builder
```

## コードアーキテクチャ

### アプリケーション構造
これは3プロセス構成のElectronデスクトップアプリケーションです：

- **メインプロセス** (`src/main.js`) - ファイルシステム操作を処理するNode.jsバックエンド
- **レンダラープロセス** (`src/renderer.js`) - フロントエンドUIロジックとイベントハンドリング
- **プリロードスクリプト** (`src/preload.js`) - メインとレンダラー間のセキュアなIPCブリッジ

### 主要コンポーネント

**メインプロセス (`src/main.js`)**
- 再帰的ディレクトリトラバーサルによるファイルスキャン (`scanDirectory()`)
- 時間ベースフィルタリング（時間/カスタム日時）と除外パターン
- 構造保持オプション付きファイル収集
- IPCハンドラー: `select-folder`, `get-modified-files`, `collect-files`

**フロントエンド (`src/index.html` + `src/renderer.js`)**
- フォルダ選択とファイル管理用の日本語UI
- IPCリスナーによるリアルタイム進捗追跡
- チェックボックスとメタデータ表示付きファイル選択インターフェース
- 3段階ワークフロー: スキャン → 選択 → 収集

**セキュリティモデル**
- `contextBridge` APIによるコンテキスト分離有効
- レンダラーからの直接Node.jsアクセス禁止
- すべてのファイル操作はセキュアなIPC経由

### 主要機能
- スマートファイル除外（システムファイル、パターン、ユーザー定義）
- 自動リネームによる重複処理
- 収集操作中の進捗追跡
- フォルダ構造保持オプション
- 全体的な日本語ローカライゼーション

### ファイル処理ロジック
1. **スキャン**: mtime フィルタリング付き再帰的ディレクトリトラバーサル
2. **除外**: 複数層（システムファイル、ユーザーパターン、隠しファイル）
3. **収集**: 進捗レポート付きアトミックコピー操作
4. **競合解決**: 重複時の自動ファイルリネーム

### UI状態管理
すべての状態は `renderer.js` のグローバル変数に保存：
- `sourceFolder`, `destFolder` - 選択されたディレクトリ
- `modifiedFiles` - スキャン結果配列
- `selectedFiles` - ユーザー選択ファイルパスのSet
- パフォーマンス向上のため `elements` オブジェクトにDOM要素をキャッシュ
